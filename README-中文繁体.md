# 自動化巡檢管理平台

## 📖 簡介

本平台是一個面向金融機構及複雜 IT 系統的**自動化巡檢管理平台**，用於替代傳統「人工戰術」式的系統巡檢。透過腳本化、編排化的自動任務，對伺服器、資料庫、中介軟體、網路設備等資源進行持續監測，自動產生巡檢報告並發出升級通知，顯著提升維運效率、降低漏檢/錯檢風險，保障系統安全與穩定。

## 🎯 背景與痛點

- **管理效率低下**：重複性工作多，人力消耗大，容易出現漏檢、錯檢。
- **安全風險高**：金融系統是攻擊目標，人工巡檢難以即時發現設定變更、憑證過期、連接埠異常等隱患。
- **資料分析不足**：傳統方式缺乏對巡檢資料的挖掘，無法提前發現潛在故障。

本平台透過自動化技術實現設備持續監測、快速識別問題並採取應對措施，最大化系統可用性。

## ✨ 核心功能

| 功能模組 | 說明 |
|---------|------|
| **資源管理** | 批次匯入/手動新增應用系統、中介軟體、網路設備等巡檢資源 |
| **作業管理** | 提供本地及遠端巡檢腳本，支援語法檢查、敏感關鍵字檢查 |
| **編排管理** | 依腳本順序對同一資源內的巡檢過程進行視覺化編排 |
| **巡檢任務** | 支援手動觸發或系統自動定時巡檢，結果登記與通知 |
| **巡檢報告** | 系統自動產生各資源的詳細巡檢報告 |
| **通知公告** | 發現憑證、應用系統等需升級時自動發出升級通知 |

## 🖥️ 支援的資源類型

### 平台/作業系統
- AIX、Windows、Linux（Redhat/麒麟等）、UnixWare、HP-Unix、AS400
- 硬體、網路設備、儲存、虛擬化、雲平台

### 資料庫與中介軟體
- **資料庫**：MySQL、Oracle、DB2、PostgreSQL、Redis
- **中介軟體**：WebLogic、WAS、MQ、Kafka、Nginx、Tomcat、IHS

### 自動化作業類型
- **腳本作業**：Shell、bat、Python
- **檔案作業**：新增刪除修改查詢、MD5校驗、解析
- **資料庫作業**：SQL、預存程序執行
- **FTP作業**：常用FTP操作

## 🔍 典型檢查項目示例

- **作業系統**：profile、hosts、sudoers、passwd、group、systeminfo、修補程式版本、磁碟/交換空間、邏輯卷、雙機服務、網路連線、系統效能、設定檔變更
- **資料庫**：版本、行程狀態、應用連線、實例設定、HADR狀態
- **中介軟體**：設定檔（如 httpd.conf、plugin-cfg.xml）、服務狀態、連接埠監聽、日誌檢查
- **網路設備**：CPU/記憶體使用率、設備鄰接關係、路由表、日誌告警、堆疊/HA狀態、溫度/電源/風扇參數
- **負載平衡器/防火牆**：運作狀態、版本資訊、設定備份、NAT連線數、備援協定狀態

## 📊 巡檢儀表板與腳本編排

- 提供視覺化巡檢儀表案例圖，全面掌握系統健康度
- 支援巡檢腳本編寫與除錯（案例圖參考）
- 支援巡檢任務的彈性編排，依序組合執行多種檢查
